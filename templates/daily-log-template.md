# Daily Log — YYYY-MM-DD

_One file per day. Morning at the top, evening at the bottom. Keep it in the same folder as your north-star doc._

---

## 🌅 Morning Intent (~3 min)

**Today I'm working on:**

> [The concrete thing. A task, not a theme. "Wire up the auth callback" not "work on auth."]

**Because:**

> [How does this connect to the "Now" box on the north-star roadmap? If you can't answer this in one line, pause and look at the doc before continuing.]

**Done looks like:**

> [Some observable end-state. "Tests pass and I can log in with Google" — not "make progress on auth."]

**Most likely derailment:**

> [What's the thing that, if it happened, would eat the day? Naming it makes it easier to notice. Optional on calm days; essential on busy ones.]

## 📌 Promised for later

_Forward-promises captured throughout the day — pulled in by the next `/start-day`. Each item: `- [YYYY-MM-DD or "later"] — what you said you'd do`. Leave the section empty if nothing came up; the agent will append items as they're mentioned._

---

### Notes & scratch

_Anything else — links, half-thoughts, frustrations, code snippets, screenshots. No structure required._

---

### How this connects to the rest of the system

- **Morning intent** → keeps today aligned with the north-star doc.
- **Evening reflection** → captures raw material for Friday's post.
- **Friday weekly review** → read the week's daily logs in one sitting. Pull the best "what I learned" bullets into the north-star doc. The arc you see across 5 days is usually the post.
